# 游戏资源预览工具 — 产品需求文档（PRD）

**文档版本**：v1.0  
**创建日期**：2026-03-20  
**产品名称**：GameAssetViewer  
**目标引擎**：Cocos Creator（首要支持），通用游戏资源文件（扩展方向）

---

## 1. 背景与目标

### 1.1 背景

Cocos Creator 游戏项目通常包含数千个资源文件，分布在大量目录中，类型繁杂（图片、图集、音频、骨骼动画、字体等）。开发者在日常工作中频繁需要：

- 快速定位和确认某个资源的视觉效果
- 检索特定名称或类型的资源
- 核查骨骼动画的动画列表与播放效果
- 了解图集（Plist/TexturePacker）中包含的子 Sprite

目前这些操作必须在 Cocos Creator 编辑器内完成，操作繁琐、启动慢，且无法脱离引擎环境独立工作。

### 1.2 目标

构建一个**纯前端网页工具**，无需后端服务、无需安装，在浏览器中直接读取本地资源文件夹，提供：

- 全类型资源的可视化预览
- 快速检索与过滤
- 骨骼动画（DragonBones / Spine）的实时播放
- 良好的性能体验（支持 8000+ 文件项目）

---

## 2. 用户群体

| 用户 | 诉求 |
|---|---|
| 游戏开发者 | 快速预览图片、动画资源，确认资源是否符合预期 |
| 游戏美术 | 核查提交的资源效果，验证图集切割是否正确 |
| 游戏策划 | 查阅项目中已有的音效、动画资源，避免重复创作 |

## 3. 功能范围

### 3.1 支持的资源类型

| 编号 | 类型 | 文件格式 | 优先级 |
|---|---|---|---|
| T1 | 图片 | `.png` `.jpg` `.jpeg` `.webp` | P0 |
| T2 | 音频 | `.mp3` `.ogg` `.wav` | P0 |
| T3 | Plist 图集 | `.plist` + `.png` | P0 |
| T4 | DragonBones 骨骼动画 | `*_ske.json` + `*_tex.json` + `*_tex.png` | P0 |
| T5 | Spine 骨骼动画 | `*.json`（Spine 格式）+ `*.atlas` + `*.png` | P0 |
| T6 | TTF 字体 | `.ttf` `.otf` `.woff` `.woff2` | P1 |
| T7 | FNT 位图字体 | `.fnt` + `.png` | P1 |

**不支持**：`.prefab`、`.anim`、`.js`、`.ts` 等代码/逻辑文件（不在预览范围内）

### 3.2 不支持资源

- Prefab（`.prefab`）：Cocos 专有格式，无运行时无法渲染
- 动画剪辑（`.anim`）：依赖 Cocos 运行时
- 脚本（`.js` `.ts`）：代码文件，非视觉资源

## 4. 功能需求

### 4.1 F1 — 文件夹读取

**F1.1 主入口**
- 提供"选择资源文件夹"按钮
- 优先使用 File System Access API（`showDirectoryPicker`），支持 Chrome/Edge 86+
- 降级方案：`<input type="file" webkitdirectory>`，兼容 Firefox/Safari

**F1.2 扫描行为**
- 递归遍历所选文件夹及所有子目录
- 扫描过程显示进度（已扫描文件数/预计总数）
- 扫描结果仅读取文件元数据（路径、名称、大小），不立即读取文件内容（按需加载）
- 过滤 `.meta` 文件，不显示在资源列表中（可选：提供"显示 meta 文件"开关）
- 过滤 `.js` `.ts` `.json`（非骨骼动画的 JSON）等非视觉资源

**F1.3 资源识别**
扫描完成后自动按以下规则分类：

```
识别优先级（从高到低）：
1. DragonBones：同目录下存在 xxx_ske.json + xxx_tex.json + xxx_tex.png → 合并为一组
2. Spine：同目录下存在 xxx.atlas + xxx.json（含 skeleton 字段）+ xxx.png → 合并为一组
3. Plist 图集：xxx.plist + 同目录同名 xxx.png → 合并为一组
4. FNT 字体：xxx.fnt + 同目录关联 png
5. TTF 字体：.ttf / .otf / .woff
6. 音频：.mp3 / .ogg / .wav
7. 图片：其余 .png / .jpg（被以上类型引用的 png 不单独列出）
```

**F1.4 历史记录**
- 记录最近 5 次打开的文件夹路径（LocalStorage 存储）
- 下次可快速重新打开（File System Access API 支持持久化句柄）

---

### 4.2 F2 — 资源浏览

**F2.1 目录树**
- 左侧面板展示文件夹树形结构
- 支持展开/折叠目录节点
- 每个目录节点显示其下资源总数
- 点击目录节点，右侧展示该目录及子目录的资源

**F2.2 资源列表（右侧主区域）**
- 默认以"缩略图网格"模式展示
- 支持切换为"列表"模式（显示文件名、类型、大小、路径）
- 缩略图尺寸可调节（小/中/大，对应 80px / 120px / 180px）
- 资源卡片显示：缩略图 + 文件名 + 类型标签
- 使用虚拟滚动（Virtual Scroll），仅渲染可视区域内的卡片（支撑 5000+ 条目）

**F2.3 资源统计面板**
- 顶部/侧边显示当前目录下各类型资源数量
- 总资源数、图片数、音频数、动画数、字体数等

---

### 4.3 F3 — 搜索与过滤

**F3.1 全局搜索**
- 顶部搜索框，输入文件名关键词
- 实时过滤（300ms 防抖）
- 支持模糊匹配（不区分大小写，支持部分匹配）
- 搜索范围：文件名 + 相对路径
- 搜索结果高亮匹配文字

**F3.2 类型过滤**
- 类型筛选栏（图片 / 音频 / 图集 / DragonBones / Spine / 字体）
- 支持多选
- 与搜索联动（在已过滤类型中搜索）

**F3.3 排序**
- 按文件名排序（A-Z / Z-A）
- 按文件大小排序
- 按修改时间排序（File System Access API 支持）
- 默认按文件名升序

---

### 4.4 F4 — 图片预览器

触发方式：点击图片资源卡片

**预览功能**：
- 全屏/弹窗模式展示原图
- 透明背景用棋盘格纹理显示
- 鼠标滚轮缩放（25% ～ 400%）
- 拖拽移动
- 显示图片元信息：尺寸（宽×高）、文件大小、格式
- 上一张/下一张切换（在当前目录内）

---

### 4.5 F5 — 音频播放器

触发方式：点击音频资源卡片

**播放功能**：
- 内嵌播放器（不弹出，卡片展开或侧边展开）
- 播放/暂停、进度条拖动、音量控制
- 显示时长、文件大小、格式
- 支持循环播放开关
- 可选：Web Audio API 绘制实时频谱图（P2 功能）

---

### 4.6 F6 — Plist 图集预览器

触发方式：点击 Plist 图集资源卡片

**预览功能**：
- 左侧：完整图集大图（可缩放）
- 右侧：所有子 Sprite 列表（网格展示，显示名称 + 尺寸）
- 点击子 Sprite：高亮在大图中的位置，右侧放大展示单个 Sprite
- 显示统计：子 Sprite 总数、图集尺寸
- 支持子 Sprite 名称搜索

**解析支持**：
- Apple Plist XML 格式（`<!DOCTYPE plist`）
- TexturePacker 导出格式（`{frames: {...}, meta: {...}}`）

---

### 4.7 F7 — DragonBones 骨骼动画预览器

触发方式：点击 DragonBones 资源卡片

**运行时**：dragonBones JS runtime（版本 **5.5.x**，匹配项目数据格式）+ PixiJS 渲染后端

**预览功能**：
- Canvas 区域实时渲染骨骼动画
- 动画列表（下拉选择或按钮列表），可切换播放不同动画
- 播放控制：播放/暂停、循环切换、播放速度调节（0.25x / 0.5x / 1x / 2x）
- 显示信息：骨骼名称（armatureName）、动画列表、帧率
- 背景色可切换（透明棋盘格、白色、黑色、自定义）
- Canvas 可缩放拖拽

**文件加载流程**：
```
1. 读取 *_ske.json → 解析骨骼数据
2. 读取 *_tex.json → 解析纹理图集元数据
3. 读取 *_tex.png → 创建 Blob URL → 加载为纹理
4. 创建 PixiJS Application + DragonBones Armature
5. 列出所有 animation 名称
6. 默认播放第一个动画
```

**错误处理**：
- 版本不匹配时显示版本警告（读取 `_ske.json` 中 `"version"` 字段）
- 缺少配对文件时显示报错提示（而非静默失败）

---

### 4.8 F8 — Spine 骨骼动画预览器

触发方式：点击 Spine 资源卡片

**运行时**：spine-player（支持 Spine **3.6** 和 **3.8**，自动检测版本）

**预览功能**：
- Canvas 区域实时渲染 Spine 动画
- 动画列表切换
- 皮肤（Skin）列表切换
- 播放控制：播放/暂停、循环、速度调节
- 显示信息：动画名称列表、皮肤列表、骨骼文件版本
- 背景色调节（同 DragonBones）

**版本自动检测**：
```
读取 *.json → 检查 JSON["skeleton"]["spine"] 字段
  "3.6.x" → 加载 spine-player-3.6.js
  "3.7.x" → 加载 spine-player-3.7.js
  "3.8.x" → 加载 spine-player-3.8.js
```

**文件加载流程**：
```
1. 读取 *.json（Spine skeleton 数据）
2. 读取 *.atlas（纹理图集映射）
3. 读取 *.png → Blob URL（纹理图）
4. 初始化 spine-player，传入三个数据源
5. 列出所有动画名称和皮肤名称
```

---

### 4.9 F9 — TTF 字体预览器

触发方式：点击 TTF 字体资源卡片

**预览功能**：
- 使用 `FontFace API` 动态加载字体
- 展示预设示例文本：
  - 英文：`The quick brown fox jumps over the lazy dog`
  - 数字：`0123456789`
  - 中文（如有 CJK 字符）：`永字八法 游戏开发 字体预览`
  - 特殊字符：`!@#$%^&*()+-=`
- 支持调节字号（12 / 16 / 24 / 36 / 48 / 72 px）
- 显示字体元信息：文件名、大小、格式

---

### 4.10 F10 — FNT 位图字体预览器

触发方式：点击 FNT 资源卡片

**预览功能**：
- 解析 `.fnt` 文件（BMFont 文本格式）
- 读取关联 `.png` 纹理
- 在 Canvas 上逐字符拼接渲染示例文本
- 输入框：允许自定义输入预览文本
- 显示字体纹理图（原始 png）供查阅

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|---|---|
| 文件夹扫描（8000 文件）| < 3 秒 |
| 首屏渲染（进入目录时）| < 500ms |
| 图片缩略图加载（可视区域）| < 200ms（懒加载） |
| 骨骼动画首帧渲染 | < 1 秒 |
| 搜索响应 | < 100ms（内存索引） |
| 内存占用 | < 500MB（1000+ 图片时） |

### 5.2 浏览器兼容性

| 浏览器 | 最低版本 | 备注 |
|---|---|---|
| Chrome | 86+ | 完整功能（File System Access API） |
| Edge | 86+ | 完整功能 |
| Firefox | 111+ | 不支持 File System Access API，降级为 webkitdirectory |
| Safari | 15.2+ | 有限支持，降级为 webkitdirectory |

**目标浏览器**：Chrome/Edge（开发者工具的主流选择）

### 5.3 部署方式

- 构建产物为**单一 HTML 文件**（或含少量 JS/CSS 的静态文件夹），无需服务器
- 双击 HTML 文件即可在浏览器中打开使用
- 可选：提供 `package.json` + `npx serve` 本地服务器启动方式（解决某些 CORS 限制）
- 不需要联网

## 7. 技术架构

### 7.1 技术栈

| 层 | 技术选型 | 理由 |
|---|---|---|
| 框架 | **Vue 3** + Composition API | 响应式、生态丰富、轻量 |
| 构建工具 | **Vite** | 快速 HMR，支持单文件 HTML 产物 |
| 样式 | **UnoCSS** 或 **Tailwind CSS** | 原子化 CSS，打包体积小 |
| 虚拟滚动 | **vue-virtual-scroller** | 支撑大量卡片渲染 |
| 模糊搜索 | **Fuse.js** | 轻量（~25KB），纯前端模糊搜索 |
| DragonBones | **dragonBones.js 5.5** + **PixiJS 7** | 官方 Pixi 适配层 |
| Spine | **spine-player 3.6 / 3.8**（按需加载） | 官方自包含播放器 |
| plist 解析 | 浏览器原生 **DOMParser** | 无需额外依赖 |
| 状态管理 | **Pinia** | Vue 3 官方状态方案 |

### 7.2 核心模块

```
src/
├── core/
│   ├── scanner.js         # 文件夹扫描、资源识别、分类
│   ├── assetIndex.js      # 内存资源索引（路径→资源对象）
│   └── fileReader.js      # 封装 File System Access API + 降级方案
├── viewers/
│   ├── ImageViewer.vue    # 图片预览器
│   ├── AudioViewer.vue    # 音频播放器
│   ├── PlistViewer.vue    # 图集预览器
│   ├── DragonBonesViewer.vue  # DragonBones 播放器
│   ├── SpineViewer.vue    # Spine 播放器
│   ├── TtfViewer.vue      # TTF 字体预览器
│   └── FntViewer.vue      # FNT 位图字体预览器
├── components/
│   ├── FolderPicker.vue   # 文件夹选择入口
│   ├── DirTree.vue        # 目录树
│   ├── AssetGrid.vue      # 虚拟滚动资源网格
│   ├── AssetCard.vue      # 资源卡片（缩略图+标签）
│   ├── SearchBar.vue      # 搜索框
│   └── TypeFilter.vue     # 类型过滤器
├── stores/
│   ├── assetStore.js      # 资源列表、过滤状态
│   └── previewStore.js    # 当前预览的资源
└── App.vue
```

### 7.3 资源识别算法（核心逻辑）

```
输入：扫描得到的所有文件（路径 + 句柄）
输出：分类后的资源对象列表

Step 1: 按目录分组所有文件
Step 2: 对每个目录内的文件集合运行识别规则（按优先级）：
  - 检测 DragonBones 三元组（_ske.json + _tex.json + _tex.png）
  - 检测 Spine 三元组（.atlas + 同名.json 且含skeleton字段 + .png）
  - 检测 Plist 二元组（.plist + 同名.png）
  - 检测 FNT 二元组（.fnt + 关联.png）
  - 已被以上规则"消费"的文件，从剩余列表移除
Step 3: 剩余文件按扩展名分类（图片/音频/字体）
Step 4: 构建全局索引（uuid/path → AssetItem）
```

## 8. 针对 Cocos Creator 的特殊支持

### 8.1 Meta 文件利用

- 解析 `.meta` 文件获取资源 UUID（用于关联关系追踪）
- Plist 图集：通过 `subMetas` 字段获取子 Sprite 的 UUID
- 可选功能（P2）：通过 UUID 反查哪些 Prefab 引用了某个资源

### 8.2 资源组织模式识别

自动识别 Cocos Creator 特有的资源组织模式：
- **Trait 模块**：检测目录名以 `Trait` 结尾的目录，在 UI 上标记为"Trait 模块"
- 按 Trait 分组浏览视图（展示某个 Trait 中所有资源）

### 8.3 已验证的版本兼容

| 资源类型 | 版本 | 验证来源 |
|---|---|---|
| DragonBones | **5.5** | `BorderGemCollectSpreadTrait/dragonbones/gameplay_gemsOut_ske.json` |
| Spine | **3.6.53** | `ChapterReplaySkipGoalTrait/spine/gameplay_shuaping2.json` |
| Spine | **3.8.99** | `ClassGuideConfigEnhanceTrait/spine/gameplay_guide_eff.json` |

工具需同时支持 Spine 3.6 和 3.8（同个项目内混用），通过读取 JSON 中 `skeleton.spine` 字段自动选择对应 runtime。

## 10. 里程碑规划

### Phase 1 — 基础框架（核心可用）
- 文件夹选择与扫描
- 目录树、资源网格（虚拟滚动）
- 搜索 + 类型过滤
- 图片预览器
- 音频播放器

### Phase 2 — 图集与字体
- Plist 图集预览器（含子 Sprite 列表）
- TTF 字体预览器
- FNT 位图字体预览器
