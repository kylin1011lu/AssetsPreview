# 项目文件规范

## 临时文件管理

### ❌ 禁止出现的情况

- 根目录不得遗留 `.bak`、`.new`、`.tmp` 等备份/草稿文件
- 根目录不得放置一次性调试脚本（如 `check-build.mjs`、`check-types.mjs`）
- 调试输出文件（如 `spine_vers.txt`）不得提交到版本库

### ✅ 正确做法

| 文件类型 | 放置位置 | Git 策略 |
|---|---|---|
| 一次性辅助脚本（类型检查、构建验证等） | `tools/` | 用完即删；如有复用价值则保留并加注释 |
| 调试输出、临时数据 | `temp/` | 整个目录已被 `.gitignore` 忽略 |
| 备份文件（`.bak`）、草稿文件（`.new`） | 不保留 | 所有 `*.bak`、`*.new` 已被 `.gitignore` 忽略，操作完成后立即删除 |

### `tools/` 目录

存放**开发辅助脚本**，提交到 git。脚本命名要体现用途，并在文件顶部注释说明：

```
tools/
  check-build.cjs     # 触发 vite build 并捕获输出
  check-types.cjs     # 触发 vue-tsc --noEmit 并捕获输出
```

> 一次性脚本用完后删除；有长期复用价值的脚本才保留。

### `temp/` 目录

存放**运行时临时产物**，不提交到 git（`.gitignore` 已配置）：

```
temp/
  spine_vers.txt      # spine 版本诊断输出
  build_out.txt       # 构建日志
```

## 代码文件备份策略

在原地修改文件之前，**不需要**手动创建 `.bak` 文件——使用 git 历史即可恢复。
如因工具限制确实需要临时备份，备份完成后必须立即删除，不得提交。
